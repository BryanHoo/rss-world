# 优化移动端阅读布局

## Goal
重构阅读器在手机与平板场景下的布局和操作流，让订阅导航、文章列表、文章详情和高频操作在小屏设备上更易触达、更有层次，并保持现有桌面三栏体验不回退。

## Requirements
- 将移动端阅读器从“桌面布局压缩版”调整为更适合小屏的独立结构。
- 保留现有数据流和状态来源，不在组件内新增重复的全局状态来源。
- 移动端需要保留订阅源入口、返回文章列表、搜索、设置等核心操作。
- 移动端文章详情应具备更清晰的上下文信息与操作分区，列表态与详情态的切换需要更顺手。
- 平板布局应继续保留双栏阅读能力，但视觉层级和容器质感需要与新的移动端方向一致。
- 桌面端的三栏布局、拖拽分栏和现有交互行为不能被破坏。

## Acceptance Criteria
- [ ] 手机宽度下，阅读器具有明显优化后的顶部信息区、主操作区和内容区，不再只是单条顶栏加全屏内容。
- [ ] 手机宽度下，在未选中文章和已选中文章两种状态中，订阅入口、返回列表、搜索和设置都可顺畅使用。
- [ ] 平板宽度下，文章列表与文章详情仍能并排展示，且容器样式与整体布局更统一。
- [ ] 桌面宽度下，左右分栏宽度、拖拽分隔条和现有桌面测试行为保持通过。
- [ ] 相关 `ReaderLayout` 测试覆盖新的移动端结构，并保持关键旧行为测试有效。

## Technical Notes
- 主要修改范围限定在 `src/features/reader/ReaderLayout.tsx` 及其相邻测试。
- 复用现有 `FeedList`、`ArticleList`、`ArticleView`、`Sheet`、`designSystem` 与 store 选择器，不改动请求层。
- 移动端布局优先通过组合已有组件完成，必要时抽取少量本地辅助 UI 结构，但不引入新的全局 store。
